# AI Web Ops Monitor V3 Execution Plan

## 목표
- 운영자가 2초 내에 "지금 조치할 것"을 식별할 수 있는 모니터링 UX 완성
- 상태 판단의 근거를 시계열로 축적해 오탐/미탐 감소
- 메뉴바 앱 정체성(가볍고 즉시성 높은 운영 콘솔) 유지

## Phase 1 — Quota 시계열 기반 (완료)
- `QuotaHistoryStore` 신규 도입
- 세션 refresh 성공 시 quota 샘플 자동 기록
- quota 카드에 미니 트렌드 스파크라인 렌더링
- 저장/로드/샘플 cap 테스트 추가

## Phase 2 — 조치 우선 상황판
- 위험 세션 우선 정렬 + 원인 레이블(로그인, 차단, 한도 임계, 지연)
- 상단에 "즉시 조치" 섹션(최대 3개) 도입
- 세션별 "다음 액션" CTA 자동 추천

## Phase 3 — 추론 신뢰도 엔진
- 웹 시그널/로컬 로그/최근 사용자 입력 가중치 통합
- 상태 전환 히스테리시스(짧은 깜빡임 제거)
- "왜 이 상태인지" 설명 라인 노출

## Phase 4 — 운영 자동화/품질
- 경고 알림의 쿨다운/중복 억제 정책 강화
- 실패 재시도 전략 표준화
- 회귀 테스트(상태 매핑/스토어/핵심 화면) 확대

## 검증 규칙
- 각 Phase 종료 시 `swift test`, `./scripts/build_app.sh` 통과
- 문서/테스트 동시 업데이트
