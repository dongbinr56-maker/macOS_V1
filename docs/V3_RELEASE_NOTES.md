# AI Web Ops Monitor V3 릴리즈 노트

## 추가된 기능

### 1) Quota 트렌드 히스토리
- 세션 새로고침 때 quota progress를 시계열로 누적 저장합니다.
- 각 quota 카드에 최근 흐름을 보여주는 스파크라인을 표시합니다.
- 운영자는 "현재 수치"뿐 아니라 "감소 추세"까지 한 번에 확인할 수 있습니다.

### 2) 즉시 조치 보드
- Overview 상단에 `즉시 조치` 영역이 추가되었습니다.
- 로그인 필요, 한도 임계, 세션 지연 등 위험 세션을 우선순위로 노출합니다.
- 각 항목은 상황별 액션 버튼(`로그인`/`새로고침`/`열기`)을 제공합니다.

## 수정된 기능

### 1) 카드 액션 정책
- 1차 액션 제목이 정적이 아니라 상태 기반으로 바뀌었습니다.
  - 로그인 필요/실패: `로그인`
  - 리스크 상태: `새로고침`
  - 정상 상태: `열기`

### 2) 상태 표기 안정화
- 비핵심 상태(`working`, `waiting`, `responding`, `idle`)의 짧은 깜빡임 전환을 완화했습니다.
- 핵심 경보 상태(`needsLogin`, `blocked`, `quotaLow`, `error`, `stale`)는 즉시 반영됩니다.

### 3) 알림 노이즈 억제
- 동일 경보 키에 대해 90초 쿨다운을 적용했습니다.
- 경보 상태는 저장하되, 짧은 시간 내 반복 알림 발송은 억제합니다.

## 사용자 체감 변화
- 위험 세션을 더 빠르게 발견할 수 있습니다.
- 의미 없는 상태 깜빡임/알림 반복이 줄어듭니다.
- 사용량 감소 속도를 직관적으로 파악할 수 있습니다.
